# Project Requirements (Public Version)

This file defines the public, portfolio-safe requirements used in this repository.

## Report Deliverable

- Output format: PDF report
- Main body length: at least 6 A4 pages, 12pt
- All generated images should be placed in an appendix at the end
- Main body language: English

## Technical Scope

This repository documents an end-to-end image generation workflow with:

- ComfyUI inference workflow
- Custom LoRA training pipeline
- Single-model consistency (training and inference in one model family)

## Mandatory Analysis Sections

The report must cover:

1. **Visual comparison** across style, aesthetics, realism, and overall perception
2. **Prompt complexity comparison** from short to long prompts
3. **Limitations and future work** based on observed failure modes

## Writing Rules

- Use a clear, reproducible experiment structure
- Keep all evidence traceable via prompt IDs, seeds, and filenames
- Separate main-body analysis from appendix image catalog
